## 0. 背景與決策基準（先建立「對照組」再談改善）

現行診斷的核心結論是：**這套複雜系統全期年化 12.7%、總報酬 152%，不到 0050 買進持有（308%）的一半；
Sharpe 領先（1.16 vs 1.01）幅度小且在單一市場/單一期間樣本下不具統計顯著性；唯一站得住的硬優勢是回撤減半（-16% vs -34%）。**
因此 v2 的第一原則是：**先把「被動基準」做成正式對照組與決策門檻，再決定這套主動策略值不值得繼續維運。**

- **決策基準 A — 0050 買進持有**：已有同期數據（年化 20.0% / Sharpe 1.01 / DD -34%）。
- **決策基準 B — 0050 + 波動目標（vol-target overlay）**：把 0050 報酬用「目標年化波動（如 12–15%）」做槓桿/減碼縮放，
  作為「同樣享受被動 beta、但把回撤拉到與本策略相近」的對照。**這是更公平的對手**：若本策略連「0050+vol-target」都贏不過（風險調整後），
  則複雜的選股/籌碼/regime 疊加的存在價值就要被嚴肅質疑。

> **總 Gate（凌駕各 Phase）**：v2 任何「保留並繼續開發主動策略」的決定，最低門檻是
> **在 walk-forward out-of-sample 上，對基準 B 的資訊比率(IR) 顯著為正（見 Phase 8 的檢定設計），或回撤優勢大到足以單獨構成 mandate。**
> 否則 v2 的理性結論可能是「縮小主動部位、以被動基準為主」。

## Phase 6：交易頻率與並倉上限（拆解「低頻」真因）

**對應缺失**：#5（集中度同時綁死頻率與穩定度）、#3（脆弱性／有效持倉僅 ~5.3 檔）、並回應使用者特別點名的「加大並倉上限」。

**假說**：交易頻率低的真因是「6 格並倉上限 + 抱贏家出場」，不是選股漏斗。已知 max_positions 6→20 可把交易/年從 27.8 拉到 73.8。
**但**：單純加大並倉在現有測試中**並未改善 Sharpe、還惡化最大回撤**——所以「加大並倉」不能單獨上，必須綁定出場/配重/風險預算的配套。

### 做法（方向性）
1. **並倉上限掃描（基準實驗）**：max_positions ∈ {6, 8, 10, 12, 15, 20}，其餘參數固定，記錄 交易/年、年化、Sharpe、Calmar、DD、單檔最大貢獻度。
   - 目的是**畫出「頻率 vs 風險調整報酬」的權衡曲線**，找出 DD 不破線的前提下 Sharpe/Calmar 的轉折點，而不是直接選 20。
2. **配套 A — 縮小單檔配重 / 收緊單檔上限**：加倉位數時，同步把 `position_size_pct` 上限與 vol-target 每檔配重下調，
   讓「總風險預算」大致守恆（避免持倉數變多但每檔仍吃滿、總曝險與單檔特異風險同時放大）。
3. **配套 B — 組合層風險預算 / 波動目標**：在投組層加「目標組合波動」或「總曝險上限」，由它反推可同時持有的有效檔數，
   而不是用固定格數硬切。
4. **配套 C — 重新檢視抱贏家出場**（與 Phase 7 共用）：若放開格數但出場仍「抱到 60 天/ATR 才走」，
   週轉與資金效率未必改善；需與 Phase 7 的出場改造一起做 A/B。

### 驗證指標（Gate）
- **DD 不得劣於現況太多**：候選設定的全期最大回撤 **≤ -18%**（軟線；理想回到 ≤ -15% Gate）；**任何使 DD > -20% 的設定一律否決**。
- **風險調整報酬要有實質提升**：Calmar 或 Sharpe **相對 max_positions=6 基準提升 ≥ 10%**，否則「加頻」不算成功。
- **不得靠單檔**：單檔對總報酬貢獻度上限、前 3 大持倉貢獻佔比需下降（佐證分散度真的改善）。
- 須在 walk-forward（Phase 8）重跑，in-sample 勝出不算數。

### 風險
- **加倉位數＝放大總曝險與特異風險**：若沒做配套 A/B，極可能複製「Sharpe 沒進步、DD 變差」的已知結果。
- **過擬合風險**：在 35 檔小 universe 上掃 max_positions 仍是同一段歷史調參，需用 Phase 8 跨期/跨市場把關。
- **容量/零股**：持倉數增加在 100k 資金下會逼近「每檔配重低於零股最小可交易單位」的下限，滑價吃掉增益（見 #7）。

### 結果（2026-06-16 執行）

> 執行：`capped_sim.run_capped` **行為中性參數化**（新增 `atr_mult/atr_lo/atr_hi/max_hold/target_vol` 可選參數，
> 預設＝舊寫死值 4.5/0.08/0.09/60/0.02；中性檢查「no-arg == 顯式字面值」逐鍵相等通過，不改任何 live 行為）。
> 實驗純快取、LIVE 35 檔、100k、零股、2018–2025。notebook：`notebooks/p6_maxpos_sweep.py`、`notebooks/p6_exit_linkage.py`。
> ⚠️ 以下皆 **in-sample 單期數字＝線索**，未過 walk-forward 不得調 live（見頂部語氣聲明 + Phase 8）。
> 基準採 35-檔自建訊號路徑：固定N=6 = 年化 12.5% / Sharpe 1.14 / **DD -13.8%** / PF 2.04 / 221 筆（27.6/年）/ Calmar 0.91 / top1 18% / top3 49%
> （DD -13.8% 而非 6.2 的 -16.0%，是 6.5#3 已知的訊號建構路徑依賴，±2.2pp）。

**(1) 並倉上限 × 配重（配套A）— 證實「單純加大並倉，即使含配套A，仍不足」**

| 設定 | 交易/年 | 年化% | Sharpe | Calmar | DD% | top3% | Gate |
|---|---|---|---|---|---|---|---|
| 固定 N=6（基準）| 27.6 | 12.5 | 1.14 | 0.91 | -13.8 | 49 | — |
| 固定 N=10 | 44.0 | 11.2 | 1.00 | 0.49 | **-22.9** | 55 | REJECT |
| 固定 N=20 | 74.1 | 12.1 | 1.05 | 0.72 | -16.7 | 47 | fail |
| 守恆 N=15 | 61.4 | 10.3 | 1.23 | 0.74 | -13.8 | 36 | fail |
| 守恆 N=20 | 73.6 | 9.4 | 1.21 | 0.78 | -12.0 | 33 | fail |

- **固定配重加大並倉 → DD 爆掉**（N=8~12 達 -21~-23%，一律 REJECT）：複製已知負面結果。
- **配套A（單檔上下限 ∝ 6/N）成功壓住 DD 與集中度**（守恆 N=20：DD -12.0%、top3 49→33%、Sharpe 1.14→1.21），
  **但稀釋年化**（12.5→9.4%）→ Calmar 不升反降 → **無格通過 Gate**。
- 結論：頻率紅利（27.6→73.6/年）沒有對應的風險調整報酬紅利；**配套A 是「必要但不充分」**。

**(2) 配套A + 配套C 出場連動 — 三者合一後 in-sample 有 4 格通過 Gate**

對「DD 有餘裕」的高 N 守恆格放寬出場（`max_hold` 60→90、`atr_hi` 0.09→0.12/0.15；`atr_lo`/`atr_mult` 不變）：

|---|---|---|---|---|---|---|---|---|---|
| 守恆N=15 · 90 · 0.15 | 45 | 12.9 | **1.36** | **1.31** | -9.9 | +10 | -0 | 28 | **PASS** |

- **最佳 in-sample：守恆N=15 / max_hold=90 / atr_hi=0.15** — Sharpe 1.14→**1.36 (+19%)**、Calmar 0.91→**1.31 (+44%)**、
  DD -13.8→**-9.9%**、top3 49→**28%**、交易 27.6→**45/年**、年化持平（12.9%）。**全面優於基準**（改善以 risk-side 為主）。
- 通過的 4 格是**連續高原**（高N守恆 × max_hold≥90 × atr_hi≥0.12），鄰格（off/0.15、90/0.09）Sharpe/Calmar 也都高
  （多僅卡單一子條件），不是孤立尖點 → 略增穩健可信度，但仍 in-sample。
- **改善來自風險側不是報酬側**：Calmar 提升幾乎全來自 DD 縮小（年化幾乎持平）；更寬停損＋更長持有 → 少被洗 → 曲線更平滑。

**(3) 仍未解 / 必須誠實標註的限制**
- **缺失#4（大多頭捕獲）未解**：2024 各出場設定都只有 ~+10%（0050 +49%），放寬 ATR / 拉長 max_hold **沒有**系統性改善 2024 捕獲，
  甚至更長持有反而略降 → up-capture 問題需 Phase 7 的**結構性**做法（regime 連動停損、分批停利、加碼），非本檔的寬度/天數槓桿。
- **熊市防禦未犧牲**：2022 各格落在 -1~-5%（capitulation block_only 仍擋進場），down-capture 沒惡化。
- **過擬合風險（最大）**：72 格僅 4 格通過、且最佳格用明顯較鬆的出場（90/0.15 vs 現行 60/0.09）＝單期峰值獵取。**這正是頂部語氣聲明與 Phase 6 風險點名的情境。**

### 有條件建議（gated，**不動 live config**）
1. **方向獲得 in-sample 支持**：「加大並倉」要成立，**必須**同時綁 配套A（風險預算守恆配重）+ 配套C（放寬出場）；三者缺一退回已知負面結果。
   候選區＝**高N守恆（N≈15–20）＋ max_hold≈90 ＋ atr_hi≈0.12–0.15**。
2. **live `max_positions`（現 6）維持不變**，待 **Phase 8 walk-forward** 對候選區做 out-of-sample / 跨期穩健性檢定通過後才提案調整；先排除單期過擬合。
3. **配套B（組合層 vol-target / 總曝險上限）延後**：配套A 已能控 DD，配套B 的增量待 Phase 8 後再評估。
4. **缺失#4 移交 Phase 7**：本檔證明「寬度/天數」槓桿無法救大多頭捕獲，Phase 7 應聚焦結構性出場與加碼。
5. **容量提醒**：守恆 N=20 單檔上限降到 9%（100k → 約 3k–9k/檔），高價股零股股數少、滑價佔比升；Phase 8 須計入容量衰減。

## Phase 7：出場改造（修正「趨勢策略卻在大多頭年慘敗」）

**對應缺失**：#4（2024 0050 +49% 本策略只賺 7.2%；2020 +30% 只賺 17.5%；ATR 8–9% 停損對高波動電子/AI 太緊）、
並與 #6（核心 edge 普通）相關。

**假說**：問題不在進場，而在出場與配重——「ATR 8–9% 移動停損」在高波動 AI/電子股上太緊，一波正常回檔就被洗出、之後難追回，
使本質偏防禦/低 beta 的工具，在大多頭年無法參與主升段。

### 做法（方向性）
1. **停損寬度與 regime 連動**：在多頭/低恐慌 regime 放寬 ATR 倍數或改用「結構性停損」（如前波低點 / Chandelier exit），
   在熊市/高恐慌 regime 才收緊。對照組：固定 ATR（現況）。
2. **分批/部分停利 vs 全抱**：測試「達 N×ATR 先減碼一半、剩餘續抱」是否能在大多頭年留住更多趨勢、同時不傷整體 Calmar。
3. **加碼/金字塔（謹慎）**：對已站穩的贏家小幅加碼（受 Phase 6 風險預算約束），檢驗能否提升多頭年參與度。
4. **重新評估 `max_hold_days=60`**：在多頭年是否反而提前砍掉仍在趨勢中的部位。

### 驗證指標（Gate）
- **大多頭年參與度**：在 0050 報酬 > +25% 的年度，本策略當年報酬 / 0050 當年報酬的**捕獲率(up-capture) 顯著上升**（目標由現況的 ~0.15–0.36 提升）。
- **不犧牲防禦**：熊市年（如 2022）的 down-capture 不得明顯惡化；全期 DD 不得破 Phase 6 設定的軟線。
- **全期風險調整報酬**：Sharpe / Calmar 不低於現況。

### 風險
- **改寬停損 = 放大單筆回撤**：可能改善多頭年卻惡化熊市，需用 up/down-capture 雙指標同時把關。
- **加碼會放大集中度**：必與 Phase 6 風險預算綁定，否則回到 #3 / #5 的老問題。
- 高度**過擬合風險**：停損寬度/倍數極易針對歷史調峰值，務必 walk-forward。

## Phase 8：抗過擬合 — walk-forward 與跨市場/跨期壓力測試

**對應缺失**：#3（過度擬合與脆弱性：同 35 檔僅差 3 個訊號，DD 就從 -13.8% 擺到 -16.0%）。

**假說**：現有參數多半針對 2018–2025 台股調峰值；有效持倉僅 ~5.3 檔使權益曲線被單檔特異風險主導，前瞻穩健性可能遠低於回測。

### 做法（方向性）
1. **滾動 walk-forward**：以「訓練窗 → 驗證窗」滾動（如 4 年訓練 / 1 年前進），所有 Phase 6/7 的候選參數一律在「未見過的前進窗」評分，
   只看 out-of-sample 聚合結果。
2. **參數敏感度 / 高原檢測**：對關鍵參數（ATR 倍數、量比門檻、max_positions、max_hold_days）畫績效曲面，
   要求最終選點落在**平坦高原**而非尖峰（尖峰＝過擬合訊號）。
3. **隨機/擾動測試**：對進場日做小幅隨機延遲、對成交價加額外滑價、對 universe 做 bootstrap 重抽，檢查績效分佈是否穩定。
4. **跨市場 sanity check（可選、低優先）**：把同一套規則套到另一個市場/指數成分（如美股大型股或另一段更早的台股期間）做「邏輯是否泛化」的定性檢查——
   不是要它在他國也賺錢，而是看 edge 是否完全只存在於這段台股歷史。

### 驗證指標（Gate）
- **out-of-sample 達 Gate**：滾動前進窗聚合的 Sharpe ≥ 1.0、DD ≤ -15%、年化 ≥ 10%（與原 Gate 對齊，但量在 OOS）。
- **穩健性**：擾動測試下，年化與 DD 的分佈不應出現「中位數遠低於點估計」或「長左尾」。
- **參數高原**：選點鄰域內績效退化幅度有上限（例如 ±1 檔 max_positions / ±0.5 ATR 倍數，Sharpe 退化 < 15%）。

### 風險
- **OOS 樣本仍是同一市場同一大牛市背景**，walk-forward 可降低但無法消除「整段期間都是台股多頭」的環境偏誤。
- 跨市場測試的資料管線成本與授權問題（FinMind 僅台股），列為低優先。

## Phase 9：消除後見之明 — point-in-time 規則化動態選股

**對應缺失**：#2（選股池後見之明/倖存者偏誤：watchlist 人工挑選，2026-06 才用「近 3 年 CAGR」把已漲完的 AI 贏家加入、把落後股剔除）。

**假說**：現行 watchlist 用了未來資訊優化歷史曲線，**前瞻績效恐顯著低於回測**。要證明 edge 真實，必須改成「只用當時可得資訊」的規則化選股。

### 做法（方向性）
1. **point-in-time universe 建構**：把選股池改為「在每個歷史時點 t，僅用 t 之前可得的資料」動態決定成分
   （如流動性/市值/成交額排序、上市滿 N 年等客觀規則），**禁止任何用到 t 之後資訊的篩選**（包含「近 3 年 CAGR」這種事後贏家）。
2. **規則化動態輪動**：以可前瞻計算的因子（動量、流動性、波動）規則化納入/剔除，取代人工挑名單。
3. **重跑全套診斷**：在 point-in-time universe 上重跑 Phase 6/7/8 的所有實驗，與「人工 watchlist」版做對照，
   量化後見之明貢獻了多少績效（預期會掉一截）。
4. **倖存者偏誤處理**：universe 需納入「期間內下市/重整」標的，避免只看活下來的。

### 驗證指標（Gate）
- **去除後見之明後仍達 Gate**：point-in-time 版的 walk-forward OOS 仍滿足 Sharpe ≥ 1.0、DD ≤ -15%、年化 ≥ 10%。
- **績效衰減透明化**：明確記錄「人工 watchlist → point-in-time」的績效落差，作為前瞻折扣。
- **universe 穩定性**：動態名單的換手率合理，不因雜訊每月大換血。

### 風險
- **去掉後見之明後，alpha 很可能大幅縮水甚至消失**——這正是要驗證的，但要有「結論可能是策略不成立」的心理準備。
- point-in-time 資料工程量大（需歷史成分、下市標的），FinMind 免費層資料完整度可能不足。

## Phase 10：對 0050 的正式 alpha / 資訊比率檢定（價值主張裁決）

**對應缺失**：#1（跑輸被動基準；Sharpe 領先微小且不具統計顯著性）、#6（核心 edge 普通；籌碼僅 +0.2 Sharpe）。

**假說**：本策略相對 0050 的「超額報酬」在統計上可能不顯著，唯一穩健優勢是回撤控制。需用正式檢定裁決「值不值得維運」。

### 做法（方向性）
1. **alpha / beta 迴歸**：把策略日（或月）報酬對 0050 報酬做迴歸，估計 alpha、beta，並報告 alpha 的 t 值 / 信賴區間。
2. **資訊比率(IR) + 顯著性**：計算對基準 A（0050）與基準 B（0050+vol-target）的 IR，並用 bootstrap / Newey-West 估其顯著性。
3. **籌碼層增量檢定**：用「TA only vs TA+籌碼」對照（master Phase 3.5 已規劃），正式量化籌碼疊加的 Sharpe/IR 增量是否顯著（已知約 +0.2，疑似不顯著）。
4. **回撤優勢的價值化**：若超額報酬不顯著，量化「回撤減半」是否足以單獨構成 mandate（例如以效用函數 / 風險趨避係數表達）。

### 驗證指標（Gate）
- **alpha 顯著**：對基準 B 的年化 alpha t 值 ≥ 2（或 bootstrap 95% 區間不含 0）→ 視為「主動策略有統計可辯護的價值」。
- **籌碼層須證明自己**：若 TA+籌碼 對 TA only 的 IR 增量不顯著，則**籌碼層降級為可選模組**（甚至移除以降複雜度）。
- **裁決規則**：若以上皆不顯著且回撤優勢不足以單獨成立 → v2 結論轉向「以被動基準為主、縮小主動部位」。

### 風險
- **單一市場單一期間，統計檢定本身 power 不足**：t 值/IR 都受限於 ~8 年樣本；檢定結果要與 Phase 8 的穩健性一起讀。
- **過度資料窺探（data-snooping）**：本策略已對這段歷史調過參，事後檢定 alpha 顯著性會偏樂觀，須以 OOS 報酬做檢定。

## Phase 11：容量 / 實用性 / 維運投報比評估

**對應缺失**：#7（100k 零股容量/實用性低：最低配重 1 萬/檔、高價股買不起、零股滑價吃報酬，12.7%/年 ≈ 1.27 萬絕對獲利相對維運複雜度投報比低）。

**假說**：在 100k 零股規模下，滑價/最小交易單位/高價股不可及，會把回測上的 edge 吃掉相當比例，使整體投報比（含人力維運）偏低。

### 做法（方向性）
1. **資金規模敏感度**：在 100k / 300k / 1M 下重跑，觀察滑價與最小單位對淨報酬的侵蝕曲線（找出「策略開始有意義」的最小資金）。
2. **零股滑價模型校準**：用 `src/execution/odd_lot_fill.py` 與已量測的滑價（`slippage_logger`）回灌回測，估真實淨報酬而非理論值。
3. **高價股可及性**：量化「因高價股零股仍買不起/配重不足而漏掉的訊號」對績效的影響。
4. **維運成本對帳**：把預期絕對獲利（年化 × 資金）對比 API/部署/人力時間成本，做白話投報比結論。

### 驗證指標（Gate）
- **淨報酬韌性**：加入零股滑價校準後，年化衰減幅度有上限（例如 < 2pp），否則容量問題視為實質缺陷。
- **最小可行資金**：明確給出「達成正投報比的最小資金門檻」，供使用者決策。

### 風險
- 真實滑價隨流動性與盤口變化，回灌模型仍是估計；建議與小額實盤滑價持續對帳。
