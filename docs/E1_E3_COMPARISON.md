# E1 / E2 / E3 並排比較：MA200 Overlay 的 Whipsaw 修正三方案

> **產生方式**：多 agent workflow（3 沙盒 agent 各自建構並執行 E1/E2/E3 細網格 + 1 綜整 agent 比較）。每個沙盒**純快取、0 API**（僅 `bm.load_adjusted_0050()` 讀本地 pickle，無 fetcher 網路呼叫）、**不改任何既有檔**（src / config / 既有 notebook 的 git diff 為空），引擎邏輯本地複製、byte-identical。
> **回測窗**：full = 2018–2025（含 2016 起暖身算 MA200/ATR）；**OOS（前進）= [2022, 2023, 2024, 2025]**；2022 為重災 whipsaw 年、2018 為次要 whipsaw 年。
> **基線（退化點）**：三方案的退化參數（E1 N=1／E2 band=0%／E3 K=0）皆**逐位重現 current-live**（0050 base 100% + 跌破 MA200 留 85%、漲回回滿），三 agent 的 sanity assert 全通過（曝險 Series 逐位相等、權益曲線 max|Δ|<1e-3、n_trades 126==126）。
> **性質**：**研究假說輸入、非 live 依據**。承 `docs/EVENT_DETECTION_RESEARCH.md` §1（whipsaw 修正 E1>E2>E3 排序）與 §5 Gate、`docs/DRAWDOWN_EVENT_STUDY_2020_2022.md`（whipsaw 基線：2022 在 3.5 週約 7 次穿越、損耗 ~0.77pp；2020 損耗 ~2.1pp）。
> **紀律鐵則**：總 Gate（R5 / EVENT_DETECTION §5）未過前，**live 配置（`config/settings.yaml`：MA200 + regime_action 0.85）一律不動**。本文是 **candidate 篩選、非部署決策**。
> **survivorship 上界**：FinMind 此 stack 無下市/歷史成分、0050 為單一存續 ETF → 所有 DD/whipsaw/Sharpe 數字皆為**上界**，「降回撤」效益若有偏差是**偏樂觀**（這反而讓「防禦非 alpha」的結論更穩）。
> **R5 定論不可違**：誠實池**無顯著 alpha**；本三方案一律定位為**結構性降回撤/降換手微調**，**不得宣稱 alpha**。

## 0. 一句話總結

三方案（E1 N 日確認／E2 對稱緩衝帶/hysteresis／E3 ATR 動態帶）在各自細網格上**都是平滑高原、無鋸齒孤峰**，OOS 風險調整報酬與 current-live **統計打平（差異全埋在 δ=0.513 雜訊內）＝確認無 alpha**。三者都**乾淨達成狹義目標**：大幅砍 whipsaw（2022 穿越 7→1）、牛市幾乎零額外犧牲（2023–25 對 0050 仍只少賺 ~0.6–1.5pp，且這代價來自 0.85 留倉本身、非 whipsaw filter）、降-DD edge **不惡化反而微優**。三者**Gate 裁決皆 MARGINAL**（訴求都站得住、無惡化，但非 outperformer、對 0050 全期報酬仍輸）。**建議：E1（N 日確認）最值得帶去做正式 walk-forward**（最小侵入＝引擎改一行、唯一有外部文獻量化證據、最乾淨的因果隔離）；E2 為正交備選；E3 暫緩（最複雜、無額外效益、高 K 反噬）。

## 3. 逐方案 plateau 評估（§5.4，用各自 δ 判平滑高原 vs 鋸齒孤峰）

**共同噪聲尺度**：三 agent 皆以 current-live OOS pooled 算 Lo-2002 1 SE → **δ = 0.513**（一致）。判準：細網格 OOS Sharpe 全距 << δ 且無孤立突峰 ＝ **平滑高原**（參數對風險調整報酬惰性，無 cherry-pick 可能）。

| 方案 | OOS Sharpe 全距 | 跨度 vs δ | 落在 [峰−δ] 格數 | 判定 |
|---|---|---|---|---|
| **E1** | [0.982, 1.001] | 0.019 ≈ δ/27 | 15/15 | **平滑高原**（textbook flat，無孤峰） |
| **E2** | [0.985, 1.005]※ | 0.020 ≈ δ/25 | 12/12 | **平滑高原**（曲線水平，無可挑真峰） |
| **E3** | [0.969, 0.992] | 0.023 ≈ δ/22 | 8/8 | **平滑高原**（極平坦；K≤1.5 略高、K≥2.0 略降，仍深埋 δ 內） |

**結論**：三者全部是**平滑高原、非鋸齒孤峰**。與 R0 的病態對比鮮明（R0 K=50/100/150 → OOS 0.93/0.56/1.21 鋸齒，1.21 是不可採信的 in-sample cherry-pick）；此處三方案的�rotation參數本質平滑連續，**無 cherry-pick 風險**。**全部微峰（E1 N=4、E2 band=1.25%、E3 K=1.5）皆在 δ 內 = 雜訊，不得用以挑參數。** 唯一可說的是「此參數區在 OOS 是 plateau，需 walk-forward 才能定具體值」。

> ⚠️ **E3 高 K 尾端趨勢**：K≥2.0 起 OOS Sharpe 略降（0.97）、2023/2025 報酬微退、2020 初跌保護微退（−27.0→−27.3%）＝帶太寬後「崩盤後 close 需漲更多才回補、reduced 拖長」的可預期反噬。雖仍埋在 δ 內，**方向性提示 K 不宜取大**。

## 4. 跨 agent 一致性檢查（方法論把關）

**固定對照（基準B vol0.011 / 0050 買持 / current-live MA200-85%）三 agent 數字一致**——最大跨 agent 全距 = **0.004**（遠小於 δ=0.513），純屬顯示位數差（E1 把 0050 顯示成 0.20/1.01，E3 保留 0.1996/1.013；E1/E3 live full_sharpe 1.006 vs E2 顯示 1.01）。三者皆用相同的 `agg()`（逐字複製自 `r6_retreat_finegrid.py`）計對照，**口徑一致、無錯**。

| 對照 | full_ann | full_sharpe | full_dd | oos_sharpe | wfd | 最大跨 agent 全距 |
|---|---|---|---|---|---|---|
| current-live | 0.1888（3 方一致） | 1.006/1.01/1.006 | −0.3121（一致） | 0.982/0.98/0.982 | −0.3121（一致） | **0.004** |

**唯一需指出的口徑差異（不是錯、但影響可比性）**：**網格內** OOS Sharpe 的定義 E2 與 E1/E3 不同——
- **E1、E3**：網格 `oos_sharpe` = `agg(eq, oos=True)` 的 Sharpe（FWD 全切片日報酬，含 3 個跨年首日邊界報酬）。

因此 E2 網格 OOS Sharpe 系統性略高（基線 0.995 vs E1/E3 基線 0.982）、**不可與 E1/E3 網格逐點比大小**。**裁決把關**：此差異 (≈0.013) 仍遠小於 δ，**不改任一方的 plateau 結論、不改 MARGINAL 裁決**；且**固定對照三方一致**（E2 對照也用 `agg`），所以**跨方案比較一律以「各自相對自己基線的位移」為準**，本文表 1a 已標 ※ 提醒。建議若日後做正式 walk-forward，**三方統一採 pooled 或統一採 agg**，避免口徑混用。

## 5. Whipsaw 降幅排名（核心發現：唯一明確、單調、符合機制的效果）

> 基線（每日 MA200）：2022 flips=7（與 DRAWDOWN 研究「3.5 週約 7 次穿越」吻合＝三 agent 計數 sanity 通過）、2018 flips=11、全期執行交易 126。

| 指標 | E1（N 日確認） | E2（緩衝帶） | E3（ATR 帶） |
|---|---|---|---|
| **2022 flips 最佳** | 7→**1**（N≥4；降 86%） | 7→**1**（band≥1.25%；降 86%） | 7→**1**（K≥1.0；降 86%） |
| **2018 flips 最佳** | 11→**5**（N≥4；降 55%） | 11→**1**（band=3.5%；降 91%） | 11→**1**（K=3.0；降 91%） |
| **執行交易數最少** | 126→**100**（N=10；−21%） | 126→**97**（band=3.5%；−23%） | 126→**94**（K=3.0；−25%） |

## 9. 總結與建議

### 排名（綜合「降 whipsaw × 保牛市 × 不惡化降-DD × 改動小 × 證據強」）

| 排名 | 方案 | Gate | 一句話 |
|---|---|---|---|
| **🥇 1** | **E1 N 日確認** | MARGINAL | **最值得帶去正式 walk-forward**：唯一有外部量化文獻（Alvarez）、引擎**改一行**（最小侵入、行為中性 additive 最易過 pytest）、2022 whipsaw 同樣壓到 7→1、牛市零增本（小 N 甚至略勝基線）、無 E3 高 K 初跌反噬；非對稱實驗額外**乾淨隔離出 N_exit 才是 whipsaw 驅動**＝最高診斷價值。唯一相對弱點：對稱網格 2018 flips 只到 5（非 1），DD 改善略小（−30.4% vs E2/E3 −30.7%），皆實務可忽略。 |
| **🥈 2** | **E2 對稱緩衝帶** | MARGINAL | 正交備選（距離型，可與 E1 疊加）：whipsaw 削減最徹底（2018 也能到 1）、DD 改善與 E3 同（−30.7%）、牛市零增本、plateau 平到無可挑真峰。次於 E1 僅因需狀態機（侵入較大）、無 E1 那麼直接的文獻量化、網格口徑與 E1/E3 不一致（須統一）。 |
| **🥉 3（暫緩）** | **E3 ATR 動態帶** | MARGINAL | **淘汰/暫緩**：相對 E2 無額外效益（whipsaw/DD/牛市三項與固定帶等價），但**實作最複雜**（需 ATR）、且**高 K（≥2.0）出現可預期反噬**（OOS Sharpe 略降、2020 初跌保護微退 −0.3pp、回補拖慢）。「自適應」理論優勢在此資料**未兌現超越固定帶**。依 EVENT_DETECTION §1 排序（ATR 次之）與「先簡單後複雜」，**先做 E1/E2，E3 僅在前兩者 walk-forward 後 whipsaw 仍殘留才回補**。 |

### 建議行動

1. **帶 E1（首選）+ E2（正交對照）去做正式 walk-forward**——呼應 `r1_walkforward.py` 的 fold 結構：**每 fold 內選參數**（不可用全期 in-sample plateau 挑 N/α），pooled OOS 對基準B 看 IR、看 wfd-DD vs 0050；**統一網格 OOS Sharpe 口徑**（三方都用 pooled 或都用 agg）。
2. **E3 暫緩**，列為 E1/E2 walk-forward 後的後補（僅在殘留 whipsaw 才引入）。
3. **這是 candidate 篩選、非部署**：三方案在 in-sample 全期細掃皆為 plateau ＝**無 cherry-pick 風險、但也未證 OOS 通過總 Gate**。**total Gate（R5）未過 → live（MA200 + 0.85）一律不動。**

### 一句話結論

> 三方案都是「**對每日 MA200 基線的乾淨結構性 whipsaw-降回撤微調**」：2022 假穿越 7→1、牛市近零成本、DD 不惡化反微優——**但全部無 alpha（OOS 與 live 打平、對 0050 報酬仍輸）**。**E1（N 日確認）以最小改動 + 最強證據 + 最佳診斷價值勝出，最值得帶去 walk-forward；E3 淘汰暫緩。任何落地前 live 不動。**

## 10. Caveats（誠實聲明）

1. **survivorship 上界**：FinMind 此 stack 無下市/歷史成分股、0050 為單一存續 ETF → 所有 DD/whipsaw/Sharpe 數字皆為**上界**，降回撤效益若有偏差是**偏樂觀**；這讓「防禦非 alpha」結論**更穩、非更弱**。
2. **單一市場、單期、power 有限**：OOS 僅 2022–25 四年、僅一次 2022 熊市 + 一次 2018 chop；whipsaw 削減（2022 7→1）雖**乾淨且單調**，但建立在**少數離散 regime 轉換**上，未做 bootstrap 顯著性檢定。
3. **R5 已定誠實池無顯著 alpha**：本三方案一律是**結構性降回撤/降換手規則**，**不得宣稱 alpha**；OOS Sharpe 與 live 打平正是此定論的再確認。
4. **in-sample 峰值不可信**：三方案的名目 OOS-Sharpe 峰（E1 N=4 / E2 band=1.25% / E3 K=1.5）皆在 δ=0.513 雜訊內 = **雜訊**，**不得用以挑參數**；具體參數值**必須由 walk-forward 決定**，本文只能說「此區在 in-sample 是 plateau」。
5. **描述性、非證實**：三沙盒皆 **in-sample 全期細掃（特徵化找線索）、未做 walk-forward**；plateau 平滑只說明**無 in-sample cherry-pick 風險**，**不等於 OOS 通過總 Gate**。
6. **MA200 / reduced=0.85 / ATR22 皆結構性選參（沿用 live）**，本三實驗只掃 confirmation 窗 / 帶寬 / K，未掃這些上游參數；結論限此切片。
7. **跨 agent 網格口徑差**（§4）：E2 網格 OOS Sharpe（pooled）與 E1/E3（agg）定義不同、**不可逐點比大小**；固定對照三方一致、不受影響；裁決不變（差異 << δ）。日後須統一口徑。
8. **沙盒 0 修改 live**：三 notebook 僅本地複製曝險邏輯、引擎 byte-identical、純快取 0 API；live `vol_target_exposure` 整合屬未來通過 Gate 後事項。
